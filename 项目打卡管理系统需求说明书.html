项目打卡管理系统需求说明书

一、系统概述

项目打卡管理系统是一个基于 Web 的审计项目考勤管理平台，用于管理审计项目中各类人员（审计组长、项目经理、中介人员）的现场打卡、考勤统计、请假补卡等业务。系统支持多角色权限管理，提供甘特图可视化、打卡记录明细、请假补卡审批、考勤统计分析等核心功能。

二、用户角色

2.1 审计组长

  • 权限：查看所有项目和所有人员数据
  • 功能：项目切换、全员考勤监控、请假补卡审批、考勤统计分析
  • 数据范围：可查看负责的所有审计项目

2.2 项目经理

  • 权限：查看所管理项目的数据（不含审计组长数据）
  • 功能：项目人员考勤监控、请假补卡审批、考勤统计
  • 数据范围：仅查看自己管理的项目

2.3 中介人员

  • 权限：仅查看个人数据
  • 功能：个人打卡记录查看、请假申请、补卡申请
  • 数据范围：仅查看自己所属的单个项目

三、核心功能模块

3.1 角色切换与项目管理

3.1.1 角色切换器

位置：页面右上角

展示内容：

  • 当前角色头像（姓氏首字）
  • 当前角色姓名
  • 当前角色标签（审计组长/项目经理/中介人员）

交互方式：下拉菜单选择角色

切换效果：切换后页面数据自动过滤

3.1.2 项目信息栏

展示内容：

  • 项目名称
  • 审计组长姓名（带图标）
  • 项目经理姓名（带图标）
  • 中介机构名称（带图标）
  • 中介人员数量（带图标）

角色差异：

  • 审计组长：右上角显示项目切换下拉框（搜索框左侧）
  • 项目经理/中介人员：无项目切换功能，固定显示所属项目

3.1.3 项目搜索

  • 位置：页面右上角
  • 功能：按项目名称关键词搜索
  • 宽度：220px
  • 支持清空

3.2 项目打卡甘特图

3.2.1 功能概述

以甘特图形式展示项目成员每日打卡情况，支持按月查看，提供直观的考勤状态可视化。

3.2.2 图例说明

  • 正常：绿色，工时达标（≥8小时）
  • 加班：蓝色，工时超过8小时
  • 工时不足：橙色，有打卡但工时<8小时
  • 漏打卡：红色，上班或下班未打卡
  • 请假：紫色，当日请假
  • 休息日：灰色，周末无打卡

3.2.3 操作功能

  • 现场打卡按钮：触发打卡操作
  • 月份切换：上个月/下个月按钮，当前月份禁用"下个月"
  • 当前月份显示：格式"YYYY年MM月"

3.2.4 甘特图表格

表头：

  • 姓名列（固定列）
  • 日期列（每日一列）：显示日期和星期
  • 今日高亮：蓝色背景
  • 周末标识：灰色背景

表体：

  • 每行代表一个人员
  • 姓名显示格式：姓名（角色）
  • 每个单元格显示当日打卡状态色块

悬浮提示：

  • 显示姓名、日期、上班时间、下班时间、工时
  • 漏打卡时显示"申请补卡"按钮

3.2.5 数据过滤规则

  • 审计组长：查看所有人员
  • 项目经理：查看除审计组长外的所有人员
  • 中介人员：仅查看本人

3.3 打卡记录明细

3.3.1 功能概述

以列表形式展示详细的打卡记录，包括上下班时间、工时、打卡地点、状态等信息。

3.3.2 筛选条件

  • 姓名：输入框，支持模糊搜索
  • 日期范围：日期选择器，支持快捷选项（今天、最近一周、最近一个月、最近三个月）
  • 打卡状态：下拉选择（全部/正常/加班/工时不足/漏打卡）
  • 操作按钮：搜索、重置

3.3.3 列表字段

字段名称 显示格式 说明
姓名 姓名（角色） 显示人员姓名和角色
日期 YYYY-MM-DD 打卡日期
上班打卡时间 HH:mm 上班打卡时间
下班打卡时间 HH:mm 下班打卡时间
工时 X.Xh 当日工作时长
打卡地点 地址信息 打卡位置
打卡状态 标签显示 正常/加班/工时不足/漏打卡

打卡状态标签颜色：

  • 正常：绿色
  • 加班：蓝色
  • 工时不足：橙色
  • 漏打卡：红色

3.3.4 列宽分配

所有列使用 min-width 平均分布，表格全宽显示无右侧留白。

3.3.5 数据过滤规则

  • 审计组长：查看所有人员
  • 项目经理：查看除审计组长外的所有人员
  • 中介人员：仅查看本人

3.4 请假记录

3.4.1 功能概述

管理项目成员的请假申请、审批流程和记录查询。

3.4.2 请假申请

触发入口：甘特图中点击日期单元格"申请请假"

申请表单：

  • 姓名：自动填充当前用户
  • 请假日期：日期选择器
  • 请假类型：下拉选择（事假/病假/年假/调休）
  • 请假原因：多行文本输入

提交后状态：待审批

3.4.3 请假记录列表

列表字段：

字段名称 显示格式 说明
姓名 文本 申请人姓名
请假日期 YYYY-MM-DD 请假日期
请假类型 标签 事假/病假/年假/调休
请假原因 文本 支持溢出省略
申请时间 YYYY-MM-DD HH:mm:ss 申请提交时间
审批人 文本 审批人姓名
审批状态 标签 待审批/已通过/已拒绝
审批意见 文本 支持溢出省略
操作 按钮 审批按钮（待审批状态）

请假类型标签颜色：

  • 事假：默认色
  • 病假：红色
  • 年假：绿色
  • 调休：蓝色

审批状态标签颜色：

  • 待审批：橙色
  • 已通过：绿色
  • 已拒绝：红色

3.4.4 请假审批

审批权限：项目经理、审计组长

审批弹窗：

  • 显示请假详情（姓名、日期、类型、原因）
  • 审批结果：单选（通过/拒绝）
  • 审批意见：多行文本输入

审批后状态更新：审批完成后，列表状态自动更新

3.4.5 数据过滤规则

  • 审计组长：查看所有人员
  • 项目经理：查看所有人员
  • 中介人员：仅查看本人

3.5 补卡记录

3.5.1 功能概述

管理漏打卡人员的补卡申请、审批流程和记录查询。

3.5.2 补卡申请

触发入口：甘特图悬浮提示中"申请补卡"按钮

申请表单：

  • 姓名：自动填充
  • 补卡日期：自动填充
  • 打卡类型：单选（上班打卡/下班打卡）
  • 补卡时间：时间选择器（HH:mm）
  • 补卡原因：多行文本输入

提交后状态：待审批

3.5.3 补卡记录列表

列表字段：

字段名称 显示格式 说明
姓名 文本 申请人姓名
补卡日期 YYYY-MM-DD 补卡日期
打卡类型 标签 上班打卡/下班打卡
补卡时间 HH:mm 补卡时间
补卡原因 文本 支持溢出省略
申请时间 YYYY-MM-DD HH:mm:ss 申请提交时间
审批人 文本 审批人姓名
审批状态 标签 待审批/已通过/已拒绝
审批意见 文本 支持溢出省略
操作 按钮 审批按钮（待审批状态）

打卡类型标签颜色：

  • 上班打卡：绿色
  • 下班打卡：橙色

审批状态标签颜色：

  • 待审批：橙色
  • 已通过：绿色
  • 已拒绝：红色

3.5.4 补卡审批

审批权限：项目经理、审计组长

审批弹窗：

  • 显示补卡详情（姓名、日期、类型、时间、原因）
  • 审批结果：单选（通过/拒绝）
  • 审批意见：多行文本输入

审批通过后：补卡记录生效，甘特图状态更新

3.5.5 数据过滤规则

  • 审计组长：查看所有人员
  • 项目经理：查看所有人员
  • 中介人员：仅查看本人

3.6 打卡统计

3.6.1 功能概述

按月统计项目成员的考勤情况，提供多维度的考勤数据分析。

3.6.2 统计维度

姓名 显示格式"姓名（角色）" -

实际出勤天数 实际有打卡记录的天数 -

3.6.4 数据过滤规则

  • 项目经理：查看除审计组长外的所有人员
  • 中介人员：仅查看本人

四、界面布局规范

4.1 页面结构

  • 页面标题：左上角"项目打卡"
  • 右上角区域：
    • 项目名称搜索框（220px）
    • 项目切换下拉框（280px，仅审计组长）
    • 角色切换器
  • 项目信息栏：卡片形式，展示项目基本信息
  • 页签切换：5个页签（甘特图、打卡记录明细、请假记录、补卡记录、打卡统计）
  • 内容区域：根据页签显示对应内容

4.2 卡片样式

  • 无边框：border: none
  • 无阴影：box-shadow: none
  • 圆角：border-radius: 12px
  • 卡片间距：16px

4.3 项目信息栏

布局：横向排列，flex 布局

内容：

  • 项目名称（加粗，15px）
  • 分隔线（竖线）
  • 审计组长（带图标）
  • 项目经理（带图标）
  • 中介机构（带图标）
  • 中介人员数量（带图标）

样式：

  • 图标颜色：主题色
  • 图标大小：15px
  • 间距：24px

4.4 表格样式

  • 无竖边框：不使用 border 属性
  • 固定表头：height="100%"
  • 全宽显示：width="100%"
  • 列宽：使用 min-width 平均分布

五、交互规范

5.1 角色切换

  1. 点击右上角角色切换器
  2. 下拉菜单显示所有可切换角色
  3. 选择角色后页面数据自动过滤
  4. 当前角色显示勾选图标

5.2 项目切换（审计组长）

  1. 点击右上角项目下拉框
  2. 选择项目后页面数据自动切换
  3. 项目信息栏同步更新

5.3 月份切换

  1. 点击"上个月"/"下个月"按钮
  2. 甘特图数据自动更新
  3. 当前月份禁用"下个月"按钮

5.4 打卡操作

  1. 点击"现场打卡"按钮
  2. 弹出打卡确认提示
  3. 打卡成功后甘特图实时更新

5.5 请假/补卡申请

  1. 点击对应入口打开申请弹窗
  2. 填写表单信息
  3. 提交后显示成功提示
  4. 列表自动刷新

5.6 审批操作

  1. 点击"审批"按钮打开审批弹窗
  2. 选择审批结果并填写意见
  3. 提交后显示成功提示
  4. 列表状态自动更新

六、数据规则

6.1 工时计算

  • 正常：≥8小时
  • 加班：>8小时
  • 工时不足：<8小时且有打卡记录
  • 漏打卡：上班或下班未打卡

6.2 考勤状态优先级

  1. 请假（最高优先级）
  2. 休息日
  3. 漏打卡
  4. 工时不足
  5. 加班
  6. 正常

6.3 数据过滤规则

  • 审计组长：所有数据
  • 项目经理：排除审计组长的数据
  • 中介人员：仅本人数据

6.4 项目数据范围

  • 审计组长：负责的所有项目
  • 项目经理：管理的项目（managerId 匹配）
  • 中介人员：所属的单个项目（projectId 固定）

七、技术实现

7.1 技术栈

  • 前端框架：Vue 3 + TypeScript
  • UI 组件库：Element Plus
  • 构建工具：Vite
  • 状态管理：Pinia
  • 路由：Vue Router

7.2 数据模拟

  • 使用 Mock.js 模拟后端数据
  • 固定数据，不使用随机数据
  • API 延迟 300ms 返回结果

7.3 路由配置

  • 路径：/checkin/statistics
  • 组件：src/views/checkin/statistics.vue
  • 菜单标题：menus.checkin.statistics

7.4 API 接口

接口名称 说明
getCheckinStatistics 获取打卡统计数据
getCheckinRecords 获取打卡记录明细
getLeaveRecords 获取请假记录
getMakeupRecords 获取补卡记录
applyLeave 申请请假
applyMakeup 申请补卡
approveLeave 审批请假
approveMakeup 审批补卡
checkin 现场打卡

八、非功能性需求

8.1 性能要求

  • 页面加载时间：<2秒
  • 角色切换响应时间：<500ms
  • 表格数据渲染：<1秒

8.2 兼容性要求

  • 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
  • 分辨率：1920x1080 及以上

8.3 安全要求

  • 角色权限严格控制
  • 数据过滤在前端和后端双重验证
  • 敏感操作需要二次确认

8.4 可用性要求

  • 界面简洁直观
  • 操作流程清晰
  • 错误提示友好
  • 支持键盘快捷操作
