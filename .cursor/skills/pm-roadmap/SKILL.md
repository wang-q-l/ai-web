# 产品路线图规划

## 工作流程

### Step 1: 采集规划信息

用 AskUserQuestion 工具收集：
- 规划周期（季度/半年/年度）？
- 产品当前阶段（0到1 / 1到10 / 成熟期）？
- 已确定的战略目标或OKR？
- 资源约束（团队规模、研发人天）？
- 是否有已承诺给客户/业务的功能？

### Step 2: 确定路线图维度

**按时间轴维度**（最常见）：
```
Now（当前迭代，1-4周）→ Next（下个迭代，1-3个月）→ Later（未来规划，3个月+）
```

**按主题维度**（战略规划时）：
```
增长主题 / 留存主题 / 变现主题 / 基础设施主题
```

**按版本维度**（发版驱动时）：
```
v1.0 → v1.1 → v2.0
```

### Step 4: 路线图沟通版本

根据受众生成对应版本：
- **面向研发团队**：包含技术细节、工时、依赖
- **面向管理层**：聚焦战略目标、里程碑、资源
- **面向客户/销售**：只展示功能大方向，不含时间承诺

## 最佳实践

- **Now/Next/Later** 优于固定日期路线图（更灵活，减少承诺压力）
- 路线图是沟通工具，不是合同，保持更新频率（至少每季度一次）
- 明确区分"已承诺"和"计划中"功能
- 保存至 `docs/roadmap-[产品名]-[年度].md`
