# 汇报材料生成

## 支持的汇报类型

| 类型         | 受众          | 核心诉求           |
| ------------ | ------------- | ------------------ |
| 产品月报     | 团队/直属上级 | 进展透明、问题预警 |
| 季度复盘     | 管理层        | 战略对齐、资源申请 |
| 产品立项汇报 | 决策层/投资人 | 机会确认、资源审批 |
| 需求评审     | 研发/设计团队 | 需求对齐、排期确认 |
| 产品数据周报 | 团队/运营     | 数据透明、行动协调 |

## 工作流程

### Step 1: 确认汇报场景

用 AskUserQuestion 收集：

- 汇报受众是谁（层级/角色）？
- 汇报类型和时间？
- 有哪些核心要传递的信息？
- 有没有需要特别说明的敏感内容或风险？

## 汇报技巧

**金字塔原理：**

1. 先给结论（最重要的信息）
2. 再给论据（支撑结论的数据/事实）
3. 最后给细节（如需要）

**汇报禁忌：**

- 不要用"还不错"、"基本完成"等模糊表述，用数字说话
- 不要隐藏问题，主动暴露风险比被追问强
- 不要只报喜不报忧，缺少问题分析的汇报不可信

**结构化表达：**

- 现状 → 问题/成就 → 原因 → 行动
- 数字说明结果，故事说明影响
