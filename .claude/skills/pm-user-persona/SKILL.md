# 用户画像设计

## 工作流程

### Step 1: 信息采集

用 AskUserQuestion 工具收集：

- 产品是什么？解决什么问题？
- 已有哪些用户数据（访谈/问卷/埋点/客服记录）？
- 是否有初步的用户群体假设？

### Step 2: 用户分层

首先识别用户层级：

```
核心用户（Core User）：使用频率最高，产品价值感知最强
次级用户（Secondary User）：偶尔使用，有特定场景需求
边缘用户（Peripheral User）：极少使用，但可能成为增长来源
```

### Step 3: 输出用户画像卡片

每个 Persona 包含：

```markdown
## Persona: [给画像起一个代表性的名字，如"小白领王晓"]

### 基础信息

- **年龄**：[范围，如 25-32岁]
- **性别**：[男/女/不限]
- **职业**：[具体职业]
- **收入**：[月收入范围]
- **地域**：[城市层级/地区]
- **教育**：[学历]

### 行为特征

- **设备使用**：[手机/PC/平板比例]
- **使用场景**：[何时何地使用]
- **使用频率**：[每天/每周/偶尔]
- **典型行为路径**：[从触发到完成的步骤]

### 心理特征

- **核心动机**：[驱动使用产品的根本原因]
- **期望与目标**：[希望通过产品达成什么]
- **痛点**：[当前遇到的核心问题，按严重程度排序]
  1. [最痛的问题]
  2. [次要问题]
  3. [潜在问题]
- **爽点**：[什么会让他们特别满意/惊喜]

### 决策因素

- **产品选择考虑**：[做决策时最在意的3个因素]
- **付费意愿**：[高/中/低，原因]
- **推荐行为**：[是否愿意推荐，为什么]

### 引用语（用第一人称模拟用户心声）

> "[一句代表性的用户心声]"

### 对应功能需求

| 用户需求 | 功能建议   | 优先级   |
| -------- | ---------- | -------- |
| [需求1]  | [功能描述] | 高/中/低 |
| [需求2]  | [功能描述] | 高/中/低 |
```

### Step 4: 用户旅程地图（可选）

当用户需要旅程地图时，输出：

```markdown
## 用户旅程地图：[Persona名称] × [核心场景]

| 阶段     | 认知 | 考虑 | 使用 | 留存 | 传播 |
| -------- | ---- | ---- | ---- | ---- | ---- |
| 用户行为 |      |      |      |      |      |
| 接触点   |      |      |      |      |      |
| 情绪曲线 | 😐   | 🤔   | 😊   | 😍   | 🤩   |
| 痛点     |      |      |      |      |      |
| 机会点   |      |      |      |      |      |
```

### Step 5: 用户价值矩阵

输出用户分层价值矩阵：

```
高价值 + 高频使用 → 核心用户：优先服务，保留为核心
高价值 + 低频使用 → 重点转化：找到唤醒触发点
低价值 + 高频使用 → 培育用户：提升变现路径
低价值 + 低频使用 → 边缘用户：低成本维护
```

## 输出规范

- 每个产品建议创建 2-4 个 Persona（核心用户1-2个，次级用户1-2个）
- Persona 需基于真实数据或合理假设，明确标注"基于[数据来源]"
- 避免创建过多 Persona 导致焦点分散
- 保存至 `docs/user-persona-[产品名].md`
