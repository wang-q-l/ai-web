## 2. 功能需求详述

### 2.1 Web管理后台

#### 2.1.1 首页（数据看板）

##### 2.1.1.1 功能需求概述

提供系统整体数据概览，让管理员快速了解平台运行状态、核心指标和待办事项。

##### 2.1.1.2 用户操作流程

1. 管理员登录后自动跳转至首页
2. 查看各项统计指标
3. 点击需要处理的指标进入对应管理页面
4. 查看图表了解趋势
5. 使用快捷操作入口快速创建内容

##### 2.1.1.3 功能设计

**统计卡片：**

- 今日XX数量：数字显示，点击跳转至列表
- 总XX数量：累计数据展示
- 待处理数量：需要处理的事项数
- 活跃用户数：今日活跃用户统计

**图表展示：**

- 近7日趋势图：折线图展示关键指标变化
- 分类分布图：饼图展示数据分布情况

**动态信息：**

- 最新公告：显示最新5条系统公告
- 待办事项：显示需要处理的事项列表

**快捷操作：**

- 【创建XX】按钮：快速创建新内容
- 【导入XX】按钮：批量导入数据

##### 2.1.1.4 页面布局设计

页面采用上下结构，高度为 calc(100vh - 140px)：

**顶部统计卡片区（flex-shrink: 0）：**

- 4个统计卡片横向排列（使用 ElRow 和 ElCol，:gutter="16"）
- 每个卡片包含：图标、数值、标题、环比增长
- ElCard组件样式：无边框、无阴影、圆角12px、padding 20px
- margin-bottom: 16px

**中部图表区（flex: 1）：**

- 左右两列布局（ElRow，:gutter="16"，6:6比例）
- 左侧：趋势折线图（ElCard组件）
- 右侧：分类饼图（ElCard组件）
- 卡片样式：无边框、无阴影、圆角12px、padding 20px
- margin-bottom: 16px

**底部动态信息区（flex-shrink: 0）：**

- 左右两列布局（ElRow，:gutter="16"，6:6比例）
- 左侧：最新公告列表（ElCard组件）
- 右侧：待办事项列表（ElCard组件）
- 卡片样式：无边框、无阴影、圆角12px、padding 20px

#### 2.1.2 XX管理

##### 2.1.2.1 功能需求概述

对XX进行增删改查、批量操作，支持多条件筛选和导入导出。

##### 2.1.2.2 用户操作流程

**查询流程：**

1. 进入XX管理页面
2. 使用筛选条件筛选数据
3. 点击【搜索】按钮
4. 查看筛选结果

**添加流程：**

1. 点击【+ 添加XX】按钮
2. 填写XX信息
3. 点击【保存】按钮
4. 系统验证数据
   - 验证通过：保存成功，显示提示，返回列表
   - 验证失败：显示错误提示，停留在当前页

**编辑流程：**

1. 点击列表中的【编辑】按钮
2. 修改XX信息
3. 点击【保存】按钮
4. 保存成功后返回列表

**删除流程：**

1. 点击列表中的【删除】按钮
2. 系统弹出确认对话框
3. 用户确认后执行删除
4. 删除成功后刷新列表

##### 2.1.2.3 功能设计

**搜索筛选：**

- 关键词搜索：支持按名称模糊搜索
- 状态筛选：下拉选择（全部/启用/停用）
- 时间范围：日期选择器，选择创建时间范围
- 【搜索】按钮：执行筛选
- 【重置】按钮：清空筛选条件

**批量操作：**

- 全选/反选：表格头部复选框
- 【批量导入】：支持Excel批量导入
- 【批量导出】：导出当前筛选结果
- 【批量删除】：删除选中的数据
- 【批量修改状态】：启用/停用选中的数据

**列表展示：** | 字段名称 | 字段标识 | 字段类型 | 显示格式 | 说明 | |---------|---------|---------|---------|------| | ID | id | number | 文本 | 系统自动生成 | | 名称 | name | string | 文本 | XX名称 | | 状态 | status | number | 标签 | 1-启用（绿色），0-停用（灰色） | | 创建时间 | createTime | string | 文本 | YYYY-MM-DD HH:mm | | 操作 | - | - | 按钮组 | 编辑、删除 |

**表单字段：** | 字段名称 | 字段标识 | 字段类型 | 长度限制 | 是否必填 | 默认值 | 验证规则 | 说明 | |---------|---------|---------|---------|---------|--------|---------|------| | 名称 | name | string | 50 | 是 | - | 非空 | XX名称 | | 描述 | description | string | 200 | 否 | - | - | XX描述 | | 状态 | status | number | - | 是 | 1 | 0或1 | 1-启用，0-停用 |

##### 2.1.2.4 页面布局设计

页面采用上下结构，高度为 calc(100vh - 140px)：

## 示例：在线考试系统

### 项目概述示例

```markdown
## 1. 项目概述

### 1.1 项目背景

随着数字化转型加速，传统考试模式在组织效率、成本控制、灵活性和数据分析方面面临挑战。为满足企事业单位、教育培训机构、认证机构对在线考试和练习的需求，特开发本平台，实现考试全流程数字化管理。

### 1.2 项目建设目标

1. 全流程数字化：实现从题库建设、智能组卷、考试安排、在线监考、智能阅卷到成绩分析和证书颁发的完整闭环
2. 多端协同：管理员通过Web后台进行高效管理，学员可通过Web端和移动App端灵活参与
3. 灵活组卷：支持固定试卷和随机抽题试卷，满足不同考试场景需求
4. 精细化管理：完善的权限控制体系，实现多层级组织架构下的数据隔离和权限管理
5. 数据驱动：通过多维数据分析，为教学改进和人才评估提供决策支持

### 1.3 项目建设范围

**包含功能：**

- Web管理后台：题库管理、试卷管理、考试管理、阅卷管理、练习管理、组织管理、权限管理、系统管理
- Web学员端：在线考试、在线练习、成绩查询、证书管理
- 移动App学员端：考试、练习、个人中心

**不包含功能：**

- 在线直播教学系统
- 社交互动功能
- 复杂的支付结算系统

### 1.4 业务流程图

主要业务流程：

1. 管理员创建题库和题目 → 2. 管理员创建试卷（固定或随机） → 3. 管理员创建考试并设置参数 → 4. 学员参加考试 → 5. 系统自动阅卷（客观题）+ 人工阅卷（主观题） → 6. 生成成绩报告和证书

### 1.5 用户角色定义

| 角色名称   | 角色描述             | 主要权限                     |
| ---------- | -------------------- | ---------------------------- |
| 系统管理员 | 系统最高权限管理者   | 所有功能权限，系统配置       |
| 题库管理员 | 题库和试卷管理者     | 题库、试卷的增删改查         |
| 考试管理员 | 考试组织和管理者     | 考试创建、监考、阅卷         |
| 学员       | 参加考试和练习的用户 | 参加考试、查看成绩、下载证书 |
```

## 编写技巧

### 1. 项目背景编写技巧

- 描述现状问题
- 说明为什么需要这个项目
- 阐述项目的价值和意义

### 2. 建设目标编写技巧

- 使用SMART原则（具体、可衡量、可实现、相关性、时限性）
- 每个目标独立成句
- 3-5个目标为宜

### 3. 功能设计编写技巧

- 使用列表清晰展示功能点
- 使用表格展示字段信息
- 标注必填项和验证规则

### 4. 页面布局编写技巧

- 描述整体布局结构（上下、左右）
- 说明各区域的高度和宽度
- 标注组件类型和样式
- 使用具体的CSS属性值

### 5. 操作流程编写技巧

- 使用编号列表
- 包含正常流程和异常分支
- 标注关键决策点
- 使用【按钮名称】标注操作按钮
