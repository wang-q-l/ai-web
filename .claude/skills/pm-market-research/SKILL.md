# 产品经理市场调研与竞品分析

## 工作流程

### Step 1: 明确调研目标

首先用 AskUserQuestion 工具澄清：

- 调研的产品/方向是什么？
- 核心目的（进入新市场 / 对标竞品 / 投资决策 / 产品迭代）？
- 已知的竞争对手有哪些？

### Step 2: 市场规模分析

输出结构：

```
## 市场规模
- TAM（总体可寻址市场）：全球/全国市场天花板
- SAM（可服务市场）：目标地区/用户群可服务的市场
- SOM（可获取市场）：现阶段可实际获取的市场份额
- 市场增速 CAGR：年复合增长率
- 核心驱动因素：3-5个推动因素
```

### Step 3: 竞品矩阵分析

对每个竞品输出：

```markdown
### [竞品名称]

- **定位**：一句话描述
- **目标用户**：核心用户群体
- **核心功能**：Top 5 功能
- **商业模式**：盈利方式（订阅/广告/交易佣金等）
- **定价策略**：免费/付费/Freemium，具体价格区间
- **优势**：3点核心竞争力
- **劣势**：3点明显不足
- **市场数据**：MAU/DAU/融资情况/营收（如有）
```

### Step 4: 竞品对比矩阵

生成对比表格：

| 维度     | 我方产品 | 竞品A | 竞品B | 竞品C |
| -------- | -------- | ----- | ----- | ----- |
| 核心功能 |          |       |       |       |
| 目标用户 |          |       |       |       |
| 定价     |          |       |       |       |
| 市场份额 |          |       |       |       |
| 技术壁垒 |          |       |       |       |
| 用户口碑 |          |       |       |       |

### Step 5: 机会点识别

```
## 市场机会与威胁

### 机会点（Opportunity）
- O1：竞品普遍存在的痛点/空白
- O2：新兴用户需求未被满足
- O3：技术/监管变化带来的新机会

### 威胁（Threat）
- T1：头部玩家的竞争压力
- T2：市场进入壁垒
- T3：替代品威胁

### 差异化建议
基于分析，建议差异化切入点：[具体策略]
```

### Step 6: 输出报告

将完整报告保存至 `docs/market-research-[产品名]-[日期].md`

## 报告模板

参考 `references/market-report-template.md` 获取完整报告模板。

## 质量标准

- 竞品分析至少覆盖3-5个主要竞品
- 市场规模需提供数据来源或推导逻辑
- 每个结论需有数据或事实支撑，避免主观臆断
- 机会点需与竞品弱点直接对应
