# 操作手册生成

## 支持的手册类型

| 类型         | 受众       | 特点                         |
| ------------ | ---------- | ---------------------------- |
| 用户操作手册 | 终端用户   | 图文步骤，通俗语言，场景导向 |
| 管理员手册   | 系统管理员 | 完整功能，配置说明，权限管理 |
| 快速入门指南 | 新用户     | 10分钟上手，只覆盖核心功能   |
| 功能说明文档 | 所有用户   | 功能索引，参考查阅型         |

## 工作流程

### Step 1: 读取上下文

优先读取（如存在）：

- `docs/spec.md` —— 需求说明书（获取功能模块结构）
- `docs/requirements.md` —— 需求文档
- 项目中已生成的前端代码（了解实际界面结构）

用 AskUserQuestion 确认：

- 手册类型（用户手册 / 管理员手册 / 快速入门 / 功能说明）？
- 主要受众（技术水平、使用场景）？
- 是否需要截图占位说明？

### Step 2: 分析功能结构

从需求说明书提取：

- 系统功能架构（菜单层级）
- 各功能模块的操作流程
- 用户角色和对应权限
- 关键业务规则和注意事项

### Step 3: 生成手册

#### 3.1.3 [核心操作2：查询]

**操作步骤：**

1. 进入 **[模块名]** 页面
2. 在搜索栏输入关键词，或选择筛选条件
3. 点击 **【查询】** 按钮，列表显示符合条件的数据
4. 点击 **【重置】** 可清空所有筛选条件

**支持的筛选条件：**

| 筛选项 | 说明 |
|--------|------|
| [字段A] | 支持模糊搜索 |
| [状态] | 下拉选择，默认"全部" |
| [时间范围] | 选择开始/结束日期 |

#### 3.1.4 [核心操作3：编辑]

1. 在列表中找到要修改的数据
2. 点击操作列的 **【编辑】** 按钮
3. 在弹窗中修改相应字段
4. 点击 **【保存】** 完成修改

> **注意**：灰色不可编辑的字段为系统自动生成，无法手动修改

#### 3.1.5 [核心操作4：删除]

1. 选中要删除的数据（单条点击操作列【删除】，多条勾选后点击【批量删除】）
2. 系统弹出确认对话框："确定要删除该数据吗？"
3. 点击 **【确定】** 完成删除

> **⚠️ 重要提示**：删除操作不可撤销，请谨慎操作！
> 已被其他功能引用的数据无法删除，需先解除关联。

## 4. 常见问题

### Q：忘记密码怎么办？
点击登录页的【忘记密码】，输入注册手机号/邮箱，按提示重置密码。

### Q：为什么某些功能按钮显示为灰色或不可见？
可能原因：您的账号没有该功能的操作权限。请联系系统管理员开通权限。

### Q：数据保存后在列表中看不到？
1. 检查当前的筛选条件，可能过滤掉了新数据
2. 点击【重置】清空筛选条件后重新查询
3. 如仍未出现，请联系技术支持

### Q：导出的文件无法打开？
建议使用 Microsoft Excel 或 WPS Office 打开 .xlsx 格式文件。

## 写作原则

**语言风格：**

- 用第二人称（"您"），口吻友好
- 操作步骤用数字列表，每步一个动作
- 界面元素用【】标注（如【保存】按钮）
- 重要提示用"注意："或"⚠️"标注

**避免：**

- 不用"点击按钮"，用"点击【按钮名称】"
- 不说"系统会自动"，说"系统将显示/提示/跳转到"

## Word 导出功能

操作手册文档可以导出为 Word 格式，使用公共文档导出工具。

### 文档要求

操作手册 Markdown 文件必须包含 YAML frontmatter：

```yaml
---
title: '操作手册'
subtitle: "[项目名称]\n\n[客户名称]"
author: '编制单位：[编制单位名称]'
date: '[年月]'
lang: zh-CN
toc: true
---
```

### 导出命令

```bash
# 导出用户操作手册
bash .claude/skills/common/doc-export/scripts/export-word.sh docs/operation-manual-user.md

# 导出管理员手册
bash .claude/skills/common/doc-export/scripts/export-word.sh docs/operation-manual-admin.md

# 导出快速入门指南
bash .claude/skills/common/doc-export/scripts/export-word.sh docs/quick-start.md
```

### 样式说明

导出的 Word 文档使用统一的样式配置，与需求说明书、测试用例保持一致。详细配置请参考：`.claude/skills/common/doc-export/README.md`

- 不省略步骤，即使看起来"显而易见"
